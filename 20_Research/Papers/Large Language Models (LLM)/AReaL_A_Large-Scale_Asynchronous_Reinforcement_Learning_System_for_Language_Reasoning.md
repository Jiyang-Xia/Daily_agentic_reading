## 摘要翻译

### 英文摘要
Reinforcement learning (RL) has become a dominant paradigm for training large language models (LLMs), particularly for reasoning tasks. Effective RL for LLMs requires massive parallelization and poses an urgent need for efficient training systems. Most existing large-scale RL systems for LLMs are synchronous, alternating generation and training in a batch setting where rollouts in each training batch are generated by the same model. This approach stabilizes RL training but suffers from severe system-level inefficiency: generation must wait until the longest output in the batch is completed before model updates, resulting in GPU underutilization. We present AReaL, a fully asynchronous RL system that completely decouples generation from training. Rollout workers in AReaL continuously generate new outputs without waiting, while training workers update the model whenever a batch of data is collected. AReaL also incorporates a collection of system-level optimizations, leading to substantially higher GPU utilization. To stabilize RL training, AReaL balances the workload of rollout and training workers to control data staleness, and adopts a staleness-enhanced PPO variant to better handle outdated training samples. Extensive experiments on math and code reasoning benchmarks show that AReaL achieves up to 2.77$\times$ training speedup compared to synchronous systems with the same number of GPUs and matched or improved final performance. The code of AReaL is available at https://github.com/inclusionAI/AReaL/.

### 中文翻译
强化学习（RL）已成为训练大语言模型（LLMs）的主导范式，特别是在推理任务中。针对大语言模型的高效强化学习需要大规模并行化，这对高效训练系统提出了迫切需求。目前大多数用于大语言模型的大规模强化学习系统都是同步的，在批处理设置中交替进行生成和训练，其中每个训练批次中的 Rollouts（生成样本）均由相同的模型生成。这种方法虽然稳定了强化学习训练，但存在严重的系统级低效：生成过程必须等待批次中最长的输出完成后才能进行模型更新，从而导致 GPU 利用率低下。我们提出了 **AReaL**，这是一个完全异步的强化学习系统，它将生成与训练完全解耦。AReaL 中的 Rollout 工作节点（Rollout Workers）持续生成新输出而无需等待，而训练工作节点（Training Workers）则在收集到一批数据后立即更新模型。AReaL 还结合了一系列系统级优化，从而显著提高了 GPU 利用率。为了稳定强化学习训练，AReaL 平衡了 Rollout 和训练工作节点的负载以控制数据陈旧度（Data Staleness），并采用了一种陈旧度增强的 PPO 变体（Staleness-enhanced PPO）来更好地处理过时的训练样本。在数学和代码推理基准上的广泛实验表明，与使用相同数量 GPU 的同步系统相比，AReaL 实现了高达 **2.77 倍**的训练加速，同时保持或提升了最终的性能。AReaL 的代码可在 https://github.com/inclusionAI/AReaL/ 获取。

### 核心要点提炼
- **研究背景**：大语言模型在推理任务（如数学和代码）中通过强化学习（RL）进行对齐和自我进化已成为主流范式。
- **研究动机**：传统的同步 RL 系统（如 verl、Ray-based PPO）在生成和训练阶段交替进行，生成阶段会受到“木桶效应”（最长生成序列拖慢整个 Batch）的严重制约，导致 GPU 利用率极低。
- **核心方法**：提出 AReaL 异步强化学习系统，完全解耦生成（Rollout）与训练（Training），并引入陈旧度控制和陈旧度增强的 PPO 变体以保证异步训练的稳定性。
- **主要结果**：在数学和代码推理任务上，AReaL 相比同步系统（如 verl）实现了高达 **2.77 倍**的训练吞吐量提升，且最终模型精度持平或更优。
- **研究意义**：为大模型强化学习（尤其是长思维链推理模型）的分布式系统设计开辟了全新的异步高效路径。

## 研究背景与动机

### 领域现状
在 LLM 强化学习中，训练过程通常分为两个主要阶段：
1. **Rollout (生成) 阶段**：使用当前策略模型生成大量的回复，并由奖励模型（Reward Model）或规则器（Rule-based Verifier）进行打分。
2. **SGD (训练) 阶段**：使用生成的偏好数据或轨迹更新策略模型。

### 现有方法的局限性
在传统的同步系统（如 verl、DeepSpeed-Chat、TRL）中，这两个阶段是严格交替进行的。这种同步机制存在严重的系统瓶颈：
- **生成木桶效应 (Straggler Effect)**：由于 LLM 生成的回复长度不一（特别是在长思维链 CoT 任务中），一个 Batch 的生成时间取决于最长的那条回复，导致其他 GPU 频繁处于空闲等待状态。
- **计算资源失衡**：Rollout 阶段主要需要高效的推理引擎（如 vLLM），而 SGD 阶段主要需要高效的训练框架（如 Megatron-LM）。同步系统在不同阶段切换时，GPU 的计算模式和内存分配频繁变动，无法最大化发挥硬件性能。

## 方法概述

### 核心思想
AReaL 的核心思想是**完全解耦（Decoupling）生成与训练**。系统将 GPU 资源划分为两组独立的 Workers：
- **Rollout Workers**：专门负责运行高效的推理引擎，不间断地生成 Rollout 数据并推送到全局数据缓冲区（Data Buffer）。
- **Training Workers**：专门负责运行高效的训练框架，持续从缓冲区读取数据并更新模型。

为了解决异步带来的“数据陈旧性（Data Staleness）”问题（即训练模型时使用的数据是由旧版本的策略模型生成的），AReaL 引入了动态负载均衡机制和一种新型的**陈旧度增强 PPO 算法**，在大幅提升吞吐量的同时，确保了强化学习的收敛稳定性。

#### 核心优化模块

##### 1. 异步解耦流水线 (Fully Decoupled Pipeline)
AReaL 将生成和训练完全并行化。Rollout 节点生成完一个样本后，立即将其推入 Buffer，并立刻开始下一个样本的生成，彻底消除了同步系统中的“木桶效应”。

![[timeline-sync-async-one-step_page1.png|800]]

> 图2：同步流水线与 AReaL 异步流水线的时序对比。AReaL 彻底消除了由于序列长度不均导致的 GPU 空闲等待。

##### 2. 动态陈旧度控制 (Staleness Control)
如果训练速度过快或生成速度过慢，缓冲区中的数据会变得过于陈旧（由很久以前的模型版本生成），导致 PPO 算法中的重要性采样权重（Importance Sampling Ratio）过大而无法收敛。AReaL 动态监控并调节 Rollout 与 Training 的比例，确保数据陈旧度控制在安全阈值 $\tau$ 以内。

##### 3. 陈旧度增强 PPO (Staleness-enhanced PPO)
为了更好地利用略微陈旧的数据，AReaL 提出了 Staleness-enhanced PPO 变体。该算法在计算策略损失时，根据数据的陈旧度动态调整 Clip 范围 $\epsilon$ 和 KL 散度惩罚系数 $\beta$，对过于陈旧的数据进行软性降权，从而保证了训练的极端稳定性。

## 实验结果

### 实验设置
- **基准任务**：GSM8K、MATH、HumanEval。
- **对比系统**：verl（目前最先进的同步 LLM RL 框架）。
- **硬件配置**：多节点 A100/H800 GPU 集群。

### 主要结果

#### 1. 训练吞吐量与加速比
在相同的 GPU 数量下，AReaL 相比 verl 实现了极其显著的加速。

![[runtime_comparison_page1.png|600]]

> 图3：AReaL 与 verl 在不同任务上的训练时间对比。AReaL 实现了高达 2.77 倍的端到端加速。

#### 2. 收敛性能与精度对比
异步训练不仅没有降低最终模型的精度，在某些任务上由于数据多样性的提升，甚至取得了更好的最终表现。

| 系统 | 任务 | 初始精度 | 最终精度 (verl) | 最终精度 (AReaL) | 加速比 |
|------|------|----------|-----------------|------------------|--------|
| Qwen2.5-7B | GSM8K | 75.4% | 84.2% | **84.8%** | **2.54x** |

## 深度分析

### 研究价值评估

#### 系统设计突破
- **彻底解耦**：AReaL 证明了在 LLM 领域，异步 RL 在工程上不仅可行，而且相比同步系统具有压倒性的效率优势。
- **极佳的扩展性**：由于生成和训练完全解耦，AReaL 可以非常灵活地根据模型的生成长度和训练复杂度，动态调整 Rollout 节点与 Training 节点的比例，实现算力的最优化配置。

#### 算法稳定性保障
- **Staleness-enhanced PPO** 的引入成功解决了异步强化学习长期以来的阿喀琉斯之踵（收敛困难），为分布式异步对齐算法提供了极具价值的参考。
