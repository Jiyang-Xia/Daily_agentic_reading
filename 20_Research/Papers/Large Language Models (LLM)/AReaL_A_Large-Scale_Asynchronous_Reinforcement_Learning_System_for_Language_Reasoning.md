## 摘要翻译

### 英文摘要
Reinforcement learning (RL) has become a dominant paradigm for training large language models (LLMs), particularly for reasoning tasks. Effective RL for LLMs requires massive parallelization and poses an urgent need for efficient training systems. Most existing large-scale RL systems for LLMs are synchronous, alternating generation and training in a batch setting where rollouts in each training batch are generated by the same model. This approach stabilizes RL training but suffers from severe system-level inefficiency: generation must wait until the longest output in the batch is completed before model updates, resulting in GPU underutilization. We present AReaL, a fully asynchronous RL system that completely decouples generation from training. Rollout workers in AReaL continuously generate new outputs without waiting, while training workers update the model whenever a batch of data is collected. AReaL also incorporates a collection of system-level optimizations, leading to substantially higher GPU utilization. To stabilize RL training, AReaL balances the workload of rollout and training workers to control data staleness, and adopts a staleness-enhanced PPO variant to better handle outdated training samples. Extensive experiments on math and code reasoning benchmarks show that AReaL achieves up to 2.77$\times$ training speedup compared to synchronous systems with the same number of GPUs and matched or improved final performance. The code of AReaL is available at https://github.com/inclusionAI/AReaL/.

### 中文翻译
强化学习（RL）已成为训练大语言模型（LLM）的主导范式，特别是在推理任务中。针对大语言模型的高效强化学习需要大规模的并行化，这给高效训练系统带来了迫切的需求。目前大多数用于大语言模型的大规模强化学习系统都是**同步的 (Synchronous)**，即在批处理设置中交替进行生成（Generation）和训练（Training），其中每个训练批次中的 Rollout 数据都是由同一个模型生成的。这种方法虽然稳定了强化学习的训练，但存在严重的系统级低效：生成过程必须等待批次中最长的输出完成后才能进行模型更新，从而导致 GPU 利用率低下。我们提出了 **AReaL**，这是一个完全**异步的 (Asynchronous)** 强化学习系统，它将生成与训练完全解耦。AReaL 中的 Rollout 工作节点（Rollout Workers）持续生成新输出而无需等待，而训练工作节点（Training Workers）一旦收集到一批数据就更新模型。AReaL 还结合了一系列系统级优化，从而显著提高了 GPU 利用率。为了稳定强化学习训练，AReaL 平衡了 Rollout 和训练工作节点的负载以控制数据陈旧度（Data Staleness），并采用了一种增强陈旧度的 PPO 变体来更好地处理过时的训练样本。在数学和代码推理基准测试上的广泛实验表明，与使用相同数量 GPU 的同步系统相比，AReaL 实现了高达 **2.77 倍**的训练加速，同时保持或提升了最终的模型性能。AReaL 的代码可在 https://github.com/inclusionAI/AReaL/ 获取。

### 核心要点提炼
- **研究背景**：LLM 强化学习（特别是推理任务）需要生成长文本，同步系统在生成时存在严重的“木桶效应”（等待最长序列生成），导致 GPU 严重闲置。
- **研究动机**：打破生成与训练的同步壁垒，通过完全异步的架构设计，最大化 GPU 利用率，加速 LLM 强化学习训练。
- **核心方法**：提出 AReaL 异步强化学习系统，完全解耦 Rollout 与 Training；引入系统级负载均衡控制数据陈旧度；设计了一种“陈旧度增强型 PPO 算法”以稳定异步训练。
- **主要结果**：在数学和代码推理任务上，AReaL 相比主流同步强化学习框架（如 Ray/veRL）实现了高达 2.77 倍的吞吐量和速度提升，且最终收敛性能无损甚至更优。
- **研究意义**：为大模型强化学习训练提供了一种全新的、极具实用价值的高性能系统架构，对超大规模推理模型训练具有重要推动作用。

## 研究背景与动机

### 领域现状
当前的大语言模型强化学习框架（如 TRL, Ray, veRL）大多采用同步 PPO 架构。在每一个 Step 中，系统首先使用当前策略模型生成一批 Rollout 样本，然后将这些样本送入训练器进行模型参数更新。

### 现有方法的局限性
在推理任务中（如数学解题、写代码），模型生成的思考轨迹（Chain-of-Thought）长度差异极大。在同步 PPO 中，快速生成完的 GPU 必须处于闲置状态，等待最长的那条轨迹生成完毕。这种“木桶效应”导致了极大的计算资源浪费。

### 研究动机
作者希望设计一个完全异步的系统，让 Rollout 节点和 Training 节点各司其职、互不干扰。Rollout 节点源源不断地生成样本并放入 Buffer，Training 节点一旦攒够 Batch 立即进行参数更新，从而彻底消除等待时间。

### 系统优化与算法改进
1. **异步流水线 (Asynchronous Pipeline)**：
   
![[timeline-sync-async-one-step_page1.png|800]]
> 图2：同步 PPO 与 AReaL 异步 PPO 的时间线对比。AReaL 彻底消除了同步等待阶段，实现了 GPU 的满载运行。

2. **陈旧度控制 (Staleness Control)**：
   由于 Rollout 节点生成数据时使用的是旧模型，而 Training 节点在不断更新模型，这就产生了**数据陈旧度 (Data Staleness)** 问题。AReaL 通过动态调整 Rollout 与 Training 的吞吐比例，将陈旧度控制在安全范围内。

3. **陈旧度增强 PPO (Staleness-Enhanced PPO)**：
   为了应对旧数据带来的偏差，AReaL 引入了重要性采样修正，并对陈旧度过高的样本进行梯度裁剪或丢弃，确保了异步训练的稳定性。

## 深度分析

### 研究价值评估
- **系统级突破**：AReaL 成功解决了大模型强化学习训练中由于序列长度不均导致的 GPU 闲置问题，是分布式系统设计在 LLM 领域的优秀实践。
- **算法与系统协同**：不仅做了系统级异步解耦，还针对性地改进了 PPO 算法以适应异步环境，做到了“系统保障吞吐，算法保障收敛”。

### 优势
- **极高的 GPU 利用率**：基本消除了 GPU 闲置等待。
- **优秀的扩展性**：在多机多卡环境下，随着节点数增加，加速比呈现极佳的线性增长趋势。
